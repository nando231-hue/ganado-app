# Migración de Almacenamiento de Animales a Supabase ## ✅ Cambios Realizados ### 1. **Inicialización de Animales** - ❌ ANTES: `let animales = JSON.parse(localStorage.getItem("animales")) || [];` - ✅ AHORA: `let animales = [];` (se carga desde Supabase tras login) ### 2. **Flujo de Login** - `checkLogin()` ahora carga animales desde Supabase después de verificar sesión - `login()` carga animales después de autenticarse exitosamente - `onAuthStateChange()` sincroniza animales cuando cambia el estado de sesión ### 3. **Funciones de Guardado Modificadas** - ✅ `agregarAnimal()` → ahora asincrónica, guarda en Supabase - ✅ `importarCSV()` → ahora asincrónica, guarda cada animal en Supabase - ✅ `importarPesos()` → ahora asincrónica, actualiza pesos en Supabase ### 4. **Funciones de Actualización Modificadas** - ✅ `editarEstado()` → ahora asincrónica, actualiza en Supabase - ✅ `editarCaravana()` → ahora asincrónica, actualiza ID en Supabase - ✅ `agregarEvento()` → ahora asincrónica, actualiza historial en Supabase - ✅ `borrarEvento()` → ahora asincrónica, actualiza historial en Supabase - ✅ `resetTiempo()` → ahora asincrónica, reseta timestamp en Supabase ### 5. **Funciones de Eliminación Modificadas** - ✅ `eliminar()` → ahora asincrónica, elimina de Supabase ### 6. **Funciones Helper Nuevas** - `cargarAnimales()` → obtiene animales del usuario desde Supabase - `guardarAnimalEnSupabase()` → inserta un animal - `actualizarAnimalEnSupabase()` → actualiza un animal - `eliminarAnimalEnSupabase()` → elimina un animal --- ## 🔧 Pasos para Implementar en Supabase ### 1. **Crear la Tabla "animales"** Ejecuta este SQL en el editor SQL de Supabase: ```sql CREATE TABLE animales ( id UUID PRIMARY KEY DEFAULT uuid_generate_v4(), usuario_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE, caravana TEXT NOT NULL, edad TEXT, raza TEXT, lote TEXT, obs TEXT, estado TEXT, lat FLOAT, lng FLOAT, peso FLOAT, historial JSONB DEFAULT '[]', ultimaActualizacion TEXT, movimientos JSONB DEFAULT '[]', pesos JSONB DEFAULT '[]', created_at TIMESTAMP DEFAULT NOW(), UNIQUE(usuario_id, caravana) ); ``` ### 2. **Crear Políticas RLS (Row Level Security)** Para que cada usuario solo vea sus animales, ejecuta: ```sql ALTER TABLE animales ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users can only see their own animals" ON animales FOR SELECT USING (auth.uid() = usuario_id); CREATE POLICY "Users can insert their own animals" ON animales FOR INSERT WITH CHECK (auth.uid() = usuario_id); CREATE POLICY "Users can update their own animals" ON animales FOR UPDATE USING (auth.uid() = usuario_id); CREATE POLICY "Users can delete their own animals" ON animales FOR DELETE USING (auth.uid() = usuario_id); ``` ### 3. **Migrar Datos (Si tienes datos en localStorage)** Si tienes datos existentes en localStorage, puedes importarlos: 1. Abre la consola del navegador en tu app 2. Corre este código para extraer los datos: ```javascript JSON.parse(localStorage.getItem("animales")) ``` 3. Luego usa el botón "📂 Importar CSV" en la app enviando los datos en formato CSV --- ## 🧪 Pruebas Recomendadas 1. **Login y Carga de Animales** - Inicia sesión - Verifica que los animales se carguen en el mapa - Abre la consola (F12) y verifica que `cargarAnimales()` se ejecute 2. **Agregar Animal** - Llena el formulario de ingreso - Haz click en "Guardar" - Verifica que aparezca en el mapa y en Supabase 3. **Editar Estado** - Click derecho en un animal - Selecciona "Editar Estado" - Verifica que se actualice en Supabase 4. **Eliminar Animal** - Click derecho en un animal - Selecciona "Eliminar" - Verifica que desaparezca tanto de la app como de Supabase 5. **Importar CSV** - Prepara un archivo CSV con datos de animales - Usa "📂 Importar CSV" - Verifica que todos se guarden en Supabase 6. **Multi-usuario** - Abre la app en otra cuenta - Verifica que vea **solo** sus animales (no los de otros usuarios) --- ## 📋 Estructura de Datos ### Animal en la App ```javascript { id: "123", // Número de caravana (identificador principal) edad: "2", raza: "Aberdeen", lote: "A1", obs: "Observaciones", estado: "Vacunar", lat: -34.5, lng: -56.2, peso: 450, historial: [...], ultimaActualizacion: "2026-04-05T...", movimientos: [...], pesos: [...], supabase_id: "uuid-...", // ID interno de Supabase usuario_id: "uuid-..." } ``` --- ## ⚠️ Notas Importantes 1. **localStorage NO se usa más para animales** - Todo va a Supabase ahora 2. **Las funciones asincrónicas** pueden tomar tiempo - La app sigue funcionando mientras se sincroniza 3. **Seguridad** - RLS asegura que cada usuario solo vea/modifique sus datos 4. **Offline** - Actualmente la app NO funciona sin conexión.
